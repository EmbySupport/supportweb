    Systemanforderungen

    Die Hardwareanforderungen für den Server hängen von mehreren Faktoren ab, darunter das Format Ihrer vorbereiteten Medien, die Art der verwendeten Client-Geräte sowie die verfügbare Bandbreite zwischen Client und Server.

    Wenn Sie Ihre Medien in einem Format vorbereiten, das alle Geräte nutzen können, wie z.B. MP4-Container mit H.264-Video und einer Stereo-AAC-Audiospur, benötigen Sie normalerweise keinen leistungsstarken Server, da alle Emby-Apps dieses Format direkt verwenden können. Wenn Sie jedoch Ihre Medien nicht im Voraus auf diese Weise vorbereiten oder nicht über ausreichende Bandbreite zwischen Client und Server verfügen, um die Medien im Originalzustand abzuspielen, wird der Emby Server Sie durch Transcodierung (Umwandlung) Ihrer Medien in Echtzeit unterstützen.

    Transcodierung ist der Prozess, bei dem Medien von einer Auflösung oder einem Format in ein anderes umgewandelt werden (z.B. erfordert das Abspielen von Full HD-Medien auf einigen Smartphones Transcodierung). Der Prozess ist CPU-intensiv, sodass ein älterer/langsamerer PC möglicherweise nicht dafür geeignet ist.

    Windows-, Mac-, Linux- oder FreeBSD-Computer

    Mindestanforderungen — keine Transcodierung

    • Intel Core 2 Duo Prozessor 1,6 GHz oder besser
    • Mindestens 1GB RAM für Windows/Mac OS X
    • Mindestens 512MB RAM für Linux
    • Windows: Vista oder neuer
    • OS X: MacOS 10.13 oder neuer
    • Ubuntu, Debian, Fedora, CentOS oder SuSE Linux

    Empfohlene Konfiguration — Transcodierung von HD-Inhalten:

    • Intel Core 2 Duo Prozessor 2,4 GHz oder besser
    • Bei Transcodierung für mehrere Geräte kann ein schnellerer CPU erforderlich sein
    • Mindestens 2GB RAM
    • Windows: Vista oder neuer
    • OS X: MacOS 10.13 oder neuer
    • Ubuntu, Debian, Fedora, CentOS oder SuSE Linux
    Note

    Emby unterstützt auch hardwarebasierte Transcodierung mit einer GPU/Grafikkarte (wenn neu genug). Dies kann die CPU-Anforderungen für mehrere Transcodierungen in Echtzeit erheblich reduzieren! Dadurch kann der Server zahlreiche Clients gleichzeitig unterstützen, ohne dass die CPU Ihres Computers überlastet wird. Einige GPUs können mehr als 20 Streams unterstützen. Das ist zusätzlich zu den Streams, die Ihr Emby-Server ohne Transcodierung verarbeiten kann!

    Netzwerkanforderungen

    Mindestanforderungen

    • Eine ADSL/Kabel/WiFi-Internetverbindung für Medien-Metadaten und Software-Updates
    • Unbelastetes WIFI mit starkem Signal über ein 802.11n-Wireless-Netzwerk ODER
    • Verdrahtetes Ethernet-Netzwerk

    Empfohlene Konfiguration

    • Eine ADSL/Kabel/WiFi-Internetverbindung für Medien-Metadaten und Software-Updates
    • 10 Megabit Upload-Bandbreite (mindestens) bei remote bereitgestellten Inhalten
    • Unbelastetes WIFI mit starkem Signal über ein 802.11AC-Wireless-Netzwerk
    • Verdrahtetes Gigabit-Ethernet-Netzwerk
