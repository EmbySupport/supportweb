    Hardware-Beschleunigung auf Windows

    Emby unterstützt die folgenden Varianten der Hardware-Beschleunigung auf Windows

    • Nvidia NVDEC & NVENC
      API zur Beschleunigung des Videoencodierens und -decodierens mit Nvidia GPUs

    • Intel QuickSync Video
      Intels Marke für dedizierte Hardware zum Videoencodieren und -decodieren

    • AMD AMF
      AMD Advanced Media Framework - Multimedia-API für AMD-Hardware zur Echtzeitverarbeitung von Multimedia

    • Microsoft DXVA
      Microsoft DirectX Video Acceleration API - Hardware-unabhängige API für hardwarebeschleunigtes Videodekodieren

    Nvidia NVDEC & NVENC

    Nvidia bietet zwei Hardware-Beschleunigungsschnittstellen an:

    • NVENC API für die Beschleunigung der Videoencodierung
    • NVDEC API für die Beschleunigung der Videodecodierung (früher NVCUVID API genannt)

    NVIDIA-GPUs enthalten einen oder mehrere hardwarebasierte Decoder und Encoder (separat von den CUDA-Kernen), die vollständig beschleunigte, hardwarebasierte Video-Dekodierung und -Encodierung für mehrere beliebte Codecs bieten. Wenn Dekodierung/Encodierung ausgelagert wird, sind der Grafikprozessor und die CPU für andere Operationen frei.

    GPU-Hardwarebeschleuniger für die Videodecodierung (NVDEC genannt) und Videoencodierung (NVENC genannt) unterstützen eine schnellere als die Echtzeit-Videobearbeitung, was sie, neben der Videowiedergabe, für Anwendungen zur Transkodierung geeignet macht.

    Unterstützte Beschleunigungen

    Die folgenden Beschleunigungen werden derzeit von Emby unterstützt. Bitte beachten Sie, dass Emby möglicherweise nicht alle vom Hardware angebotenen Beschleunigungen unterstützt und dass nicht alle Hardwaregeräte alle Beschleunigungen unterstützen.

    • Decoder
      • H.264 (AVC)
      • H.265 (HEVC)
      • MPEG2
      • MPEG4
      • VC1
      • VP8
      • VP9
    • Encoder
      • H.264 (AVC)

    Hardware-Anforderungen

    Die Hardware-Beschleunigung ist für die meisten Nvidia-Geräte ab der Kepler-Generation (z. B. GeForce GT 630) verfügbar, einschließlich sowohl Verbraucher- als auch Professional-Geräte. Detaillierte Informationen über unterstützte Hardware finden Sie in Nvidias GPU Support Matrix.

    Erforderliche Einrichtungsschritte

    Installieren Sie die neuesten Treiber für Ihre Nvidia-Hardware direkt von der Seite Nvidia Driver Downloads. Es wird dringend empfohlen, diese Treiber anstelle der mit Windows gelieferten zu verwenden. Die mindest erforderliche Treiberversion auf Windows ist 471.41
    Es ist nicht erforderlich, andere Tools oder SDKs von Nvidia zu installieren.

    Hinweise

    • Emby unterstützt den headless Betrieb für Nvidia
      Es ist nicht erforderlich, einen Monitor anzuschließen
    • Nvidia-Beschleunigung funktioniert auch, wenn Emby als Windows-Dienst läuft

    Intel QuickSync Video

    Intel® Quick Sync Video nutzt die dedizierten Medienverarbeitungskapazitäten der Intel® Graphics Technology, um schneller zu dekodieren und zu encodieren, sodass der Prozessor andere Aufgaben erledigen kann und die Systemreaktionsfähigkeit verbessert wird.

    Unterstützte Beschleunigungen

    Die folgenden Beschleunigungen werden derzeit von Emby unterstützt. Bitte beachten Sie, dass Emby möglicherweise nicht alle vom Hardware angebotenen Beschleunigungen unterstützt und dass nicht alle Hardwaregeräte alle Beschleunigungen unterstützen.

    • Decoder
      • H.264 (AVC)
      • H.265 (HEVC)
      • MPEG2
      • VC1
      • VP8
      • VP9
    • Encoder
      • H.264 (AVC)
    • Hardware-Filter
      • Skalierung
      • Deinterlacing

    Hardware-Anforderungen

    Quick Sync wurde ursprünglich in einige Sandy Bridge CPUs integriert, jedoch nicht in Sandy Bridge Pentium oder Celeron CPUs.
    Eine Übersicht über die in den verschiedenen CPU-Generationen integrierten Beschleunigungsmöglichkeiten finden Sie hier: Hardware-Dekodierung und -Encodierung. und hier GPU-Beschleunigungsmöglichkeiten.

    Erforderliche Einrichtungsschritte

    Installieren Sie die neuesten Grafiktreiber für Ihre Intel-CPU mit integrierter Grafikkarte direkt von Intels Seite: Treiber & Software. Es wird dringend empfohlen, diese Treiber anstelle der mit Windows gelieferten zu verwenden. Es ist nicht erforderlich, das Intel Media SDK zu installieren.

    Hinweise

    • Emby unterstützt den headless Betrieb (und den Betrieb als Dienst) mit Intel QuickSync nur bei Verwendung des D3D11-Codecs. Für den Normalmodus müssen Sie einen physischen Monitor an den Videoausgang anschließen.

    AMD AMF

    Das AMD Advanced Media Framework ist ein leichtgewichtiges, portables Multimedia-Framework, das die meisten plattform- und API-spezifischen Details abstrahiert und eine einfache Implementierung von Multimedia-Anwendungen unter Verwendung einer Vielzahl von Technologien wie DirectX 11, OpenGL und OpenCL ermöglicht und eine effiziente Interoperabilität zwischen ihnen erleichtert.

    Unterstützte Beschleunigungen

    Die folgenden Beschleunigungen werden derzeit von Emby unterstützt. Bitte beachten Sie, dass Emby möglicherweise nicht alle vom Hardware angebotenen Beschleunigungen unterstützt und dass nicht alle Hardwaregeräte alle Beschleunigungen unterstützen.

    • Encoder
      • H.264 (AVC)

    Hardware-Anforderungen

    Das AMF-Framework ist mit den meisten neueren Radeon-GPUs kompatibel, beginnend mit der Southern Islands-Familie und APUs der Kabini-, Kaveri-, Carrizo-Familien und neuer.

    Erforderliche Einrichtungsschritte

    Installieren Sie die neueste Radeon Software direkt von AMDs Website: AMD Treiber & Unterstützung. Es ist nicht erforderlich, das AMF SDK zu installieren. Die mindest erforderliche Softwareversion ist: AMD Radeon Software Crimson Edition 16.7.3 (16.30.2311)

    Hinweise

    • Emby unterstützt derzeit keinen headless Betrieb mit AMD AMF. Sie müssen einen physischen Monitor an den Videoausgang anschließen.
    • AMD AMF-Beschleunigung funktioniert nicht, wenn Emby als Windows-Dienst läuft.

    Microsoft DXVA

    DirectX Video Acceleration (DXVA) ist eine Microsoft-API zur Verwendung von Hardware-Beschleunigung zur Beschleunigung der Videobearbeitung. Software-Codecs und Software-Videoprozessoren können DXVA verwenden, um bestimmte CPU-intensive Operationen an die GPU auszulagern. DXVA unterstützt nur das Videodekodieren, kein Encodieren.

    Unterstützte Beschleunigungen

    Die folgenden Beschleunigungen werden derzeit von Emby unterstützt. Bitte beachten Sie, dass Emby möglicherweise nicht alle vom Hardware angebotenen Beschleunigungen unterstützt und dass nicht alle Hardwaregeräte alle Beschleunigungen unterstützen.

    • Decoder
      • H.264 (AVC)
      • H.265 (HEVC)
      • MPEG2
      • VC1

    Hardware-Anforderungen

    DXVA unterstützt Grafikgeräte von verschiedenen Anbietern, einschließlich Nvidia, Intel und AMD.

    Erforderliche Einrichtungsschritte

    DXVA ist eine Windows-Komponente und soll mit den in Windows enthaltenen Treibern funktionieren. Die Installation der neuesten Treiber von den Websites der Hersteller kann trotzdem eine gute Idee sein.

    Hinweise

    • Emby unterstützt den headless Betrieb (und den Betrieb als Dienst) nur mit D311VA-Codecs. Für einfaches DXVA2 müssen Sie einen physischen Monitor an den Videoausgang anschließen.

    Häufige Probleme und Lösungen

    • Hardware-Beschleunigung schlägt bei Remote-Desktop (RDP) unter Windows fehl

    Hinweis: Unterstützung für den headless Modus ist für die Zukunft geplant
