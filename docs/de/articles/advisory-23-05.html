    Emby Server startet nicht - Sicherheitswarnung 2023-05-25

    Anwendbarkeit

    Dieser Artikel gilt für Fälle, in denen Sie eines oder mehrere der folgenden Symptome erleben:

    • Ihr Emby Server hat sich selbst heruntergefahren und/oder startet seit dem 25.05.2023 nicht mehr
    • Sie finden folgende Nachricht im Emby Server-Protokoll: Wir haben ein bösartiges Plugin auf Ihrem System entdeckt, das wahrscheinlich ohne Ihr Wissen installiert wurde. Bitte besuchen Sie https://emby.media/support/articles/advisory-23-05.html für weitere Informationen dazu, wie Sie vorgehen sollten. Zu Ihrer Sicherheit haben wir Ihren Emby Server vorsorglich heruntergefahren Hinweis: Ihr Server wurde nie von uns direkt angesprochen. Wir nutzten unseren Standard-Update-Mechanismus
    • Bei Windows-Installationen finden Sie einen Fehler im Windows-Ereignisprotokoll, Quelle=".NET Runtime", EreignisID=1025 mit dem gleichen Text wie oben in den Ereignisdetails

    Hintergrund

    Ab Mitte Mai 2023 gelang es einem Hacker, private, von Nutzern gehostete Instanzen von Emby Server zu infiltrieren, die über das öffentliche Internet zugänglich waren und eine unsichere Konfiguration für Administrationskonten hatten. In Kombination mit der "Proxy Header Vulnerability", die kürzlich im Beta-Kanal behoben wurde, konnte ein Angreifer auf solchen Systemen administrativen Zugang erlangen. Dies erlaubte es den Angreifern schließlich, ein eigenes benutzerdefiniertes Plugin zu installieren, das eine Hintertür im laufenden Prozess von Emby Server einrichtet.

    Nach sorgfältiger Analyse und Bewertung möglicher Gegenmaßnahmen konnte das Emby-Team ein Update für Emby Server-Instanzen veröffentlichen, das das betreffende Plugin erkennen und laden verhindern kann. Aufgrund der Schwere und Natur dieser Situation und aus Vorsicht verhindern wir, dass betroffene Server nach der Erkennung erneut starten. Da die gegebene Situation direkte Maßnahmen und Beurteilung durch den Administrator erfordert, haben wir entschieden, dass das Herunterfahren des Servers und das Verhindern eines weiteren Startvorgangs die geeignetste Maßnahme ist, da es das Plugin deaktiviert, möglicherweise verhindert, dass sich die Situation verschlimmert, und gleichzeitig die Aufmerksamkeit des Administrators auf das Thema lenkt.

    Die Analyse des Plugins hat ergeben, dass es die privaten Emby Server-Anmeldedaten, einschließlich des Passworts für jeden erfolgreichen Login, an einen externen Server unter der Kontrolle der Hacker weiterleitet.

    Zu ergreifende Maßnahmen

    • Fügen Sie einen Eintrag in die etc/hosts-Datei Ihrer Server-Maschine hinzu: emmm.spxaebjhxtmddsri.xyz 127.0.0.1
      • Dies ist der Hostname des Kontrollservers, mit dem die Malware kommuniziert
      • Für Hilfe dazu, siehe bitte https://linuxize.com/post/how-to-edit-your-hosts-file/

    Überprüfen Sie Ihre Server-Maschine auf:

    • Verdächtige Nutzerkonten
    • Unbekannte Prozesse
    • Unbekannte Netzwerkverbindungen und offene Ports
    • SSH-Konfiguration
    • Firewall-Regeln
    • Ändern Sie alle Passwörter

    Emby Server starten

    Nachdem Sie die obigen Schritte durchgeführt haben, und nur dann:

    • Deaktivieren Sie den externen Netzwerkzugang
    • Gehen Sie zu folgendem Ordner im Emby Server-Datenordner: plugins/configurations
      • Finden und löschen Sie die Datei ReadyState.xml.
      • Finden und löschen Sie die Datei 'EmbyScripterX.xml' (falls vorhanden)
      • Löschen Sie die Plugin-.dll-Datei(en), die als helper.dll und/oder EmbyHelper.dll auftreten. Der Hauptspeicherort dieser Dateien ist der plugins-Ordner im Emby Server-Datenordner
    • Starten Sie den Emby Server
    • Weisen Sie allen Emby Server-Nutzern neue Passwörter zu
    • Erlauben Sie keinen lokalen Login ohne Passwort
    • Stellen Sie sicher, dass kein Nutzer ein leeres Passwort hat
    • Aktivieren Sie den öffentlichen Netzwerkzugang wieder
      • Erwägen Sie, die IP-Adresse, den Port oder den DNS-Namen zu ändern (je nachdem, was zutreffend ist)
    Note

    Wenn möglich, wäre es großartig, wenn Sie uns die Malware-dll-Datei senden könnten, damit wir analysieren können, ob verschiedene Versionen existieren.
    Bitte senden Sie die Datei per privater Nachricht in den Foren an softworkz.

    Emby Server 4.7.12 Sicherheitsupdate installieren

    • Halten Sie Ausschau nach diesem Update und installieren Sie es, sobald es verfügbar ist, es sei denn, Sie nutzen bereits Emby Beta 4.8.0.31+, das ebenfalls die notwendigen Fixes enthält.
